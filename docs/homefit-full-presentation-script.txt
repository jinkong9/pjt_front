HOME FIT 전체 발표 스크립트

발표 전제
현재 HOME FIT은 모든 기능이 완전히 고도화된 최종 버전이라기보다는, 공공데이터와 사용자 데이터를 하나의 주거 의사결정 흐름으로 연결하는 MVP 단계입니다. 그래서 발표에서는 지금 구현된 핵심 흐름과, 앞으로 확장 가능한 기술 구조를 중심으로 설명하겠습니다.


[1번 슬라이드] 표지

안녕하세요. 저희가 발표할 서비스는 HOME FIT입니다.

HOME FIT은 공공데이터 기반 통합 주거 의사결정 플랫폼입니다. 단순히 집을 검색하는 서비스가 아니라, 실거래가, 공공임대, 양도 매물, 생활권, 금융 조건을 하나의 흐름으로 연결해서 사용자가 더 합리적으로 집을 선택할 수 있도록 돕는 것을 목표로 했습니다.


[2번 슬라이드] Problem Definition

먼저 저희가 발견한 문제는 주거 정보가 너무 많이 흩어져 있다는 점입니다.

집을 구할 때 사용자는 실거래가는 따로 보고, LH 공고는 또 다른 사이트에서 확인하고, 대출 상품은 금융 사이트에서 찾아보고, 주변 생활 인프라는 지도 서비스에서 다시 검색해야 합니다.

이 과정에서 정보는 많지만 서로 연결되어 있지 않기 때문에, 사용자가 실제로 판단해야 하는 "이 집이 나에게 맞는가"라는 질문에는 바로 답하기 어렵습니다.

그래서 저희는 흩어진 주거 정보를 하나의 서비스 안에서 연결하는 방향으로 문제를 정의했습니다.


[3번 슬라이드] 기존 사용자 고충

사용자 입장에서 보면 크게 네 가지 불편함이 있습니다.

첫 번째는 정보 분산입니다. 실거래, LH, 양도, 대출 정보를 각각 다른 사이트에서 확인해야 합니다.

두 번째는 비교 단절입니다. 가격은 봤지만 생활권이 어떤지, 교통이 괜찮은지, 월 납부 부담이 어느 정도인지를 한 번에 판단하기 어렵습니다.

세 번째는 일정 누락입니다. LH 공고나 관심 매물의 신청 마감일을 사용자가 직접 기억하고 관리해야 합니다.

마지막은 단기 주거 공백입니다. 대학생 추가 합격이나 인턴처럼 빠르게 입주해야 하는 상황에서도, 적절한 단기 주거 선택지를 찾기가 쉽지 않습니다.


[4번 슬라이드] 서비스 제작 계기

저희는 집 찾기가 단순히 가격을 검색하는 과정이 아니라, 실제로 살 수 있는지 결정하는 과정이라고 생각했습니다.

하지만 기존 서비스들은 데이터가 각각 분리되어 있어서, 사용자의 상황과 조건을 기준으로 판단하기 어렵습니다.

장기 거주를 원하는 사람, LH 공공임대를 찾는 사람, 단기 거주가 필요한 사람은 각각 다른 흐름이 필요합니다.

그래서 HOME FIT은 탐색, 생활권 분석, 자금 계산, 일정 관리를 하나의 의사결정 흐름으로 연결하는 서비스를 목표로 제작했습니다.


[5번 슬라이드] 서비스 제안

HOME FIT은 공공데이터 기반 부동산 실거래 정보와 공공임대 공고, 양도 매물, 생활권 분석, 맞춤 금융 조건을 연결합니다.

사용자는 여러 플랫폼을 오가지 않고, HOME FIT 안에서 매물을 탐색하고, 주변 환경을 보고, 대출 가능성과 월 부담을 계산하고, 관심 공고나 매물을 관리할 수 있습니다.

즉, HOME FIT은 "집을 찾는 단계"부터 "실제로 선택할 수 있는지 판단하는 단계"까지 이어주는 통합 주거 의사결정 플랫폼입니다.


[6번 슬라이드] 전체 서비스 구성

전체 서비스는 네 가지 큰 기능으로 구성했습니다.

첫 번째는 실거래가 지도입니다. 아파트, 오피스텔, 원룸 등 실거래 데이터를 지도 위에서 탐색하고, 지역과 주거 유형별로 필터링할 수 있습니다.

두 번째는 LH 공공임대입니다. LH 공고 조회, 상세 정보 확인, 관심 공고 저장, 사용자 조건 기반 추천을 제공합니다.

세 번째는 양도 매물 게시판입니다. 보증금, 월세, 관리비, 양도비, 입주 가능일을 비교해서 빠른 입주가 필요한 사용자를 지원합니다.

네 번째는 생활권 분석과 금융 프로필입니다. 주변 편의시설과 교통 접근성을 분석하고, 사용자의 자산과 소득 조건을 바탕으로 대출 부담과 추천 공고를 연결합니다.


[7번 슬라이드] DB 연동 화면 01

여기서는 실제 DB와 연동된 주요 화면을 보여드립니다.

먼저 실거래 지도에서는 서울 강남구 역삼동 기준으로 실거래 데이터가 지도 마커와 함께 표시됩니다. 사용자는 지도에서 거래 위치를 보고, 선택한 매물의 상세 정보도 함께 확인할 수 있습니다.

공공임대 공고 화면에서는 LH 공고 캐시 데이터가 카드 형태로 표시되고, 지역이나 유형 필터를 통해 원하는 공고를 탐색할 수 있습니다.


[8번 슬라이드] DB 연동 화면 02

다음은 LH 캘린더와 양도 게시판입니다.

LH 캘린더는 공고일과 접수 시작 일정을 월간 캘린더 형태로 표시해서, 공고를 찾은 뒤에도 신청 일정을 놓치지 않도록 돕습니다.

양도 게시판은 실제 게시글 형태로 보증금, 월세, 입주일, 이미지 정보를 보여줍니다. 빠르게 입주할 수 있는 매물을 비교하는 데 초점을 맞췄습니다.


[9번 슬라이드] DB 연동 화면 03

생활권 분석 화면에서는 특정 지역을 기준으로 주변 시설, 버스와 지하철 접근성, 생활권 점수를 계산합니다.

마이페이지에서는 로그인 계정의 회원 정보와 관심 실거래 목록, 관심 공고, 금융 프로필 같은 개인화 데이터를 확인할 수 있습니다.

이 부분은 단순 조회에서 끝나는 것이 아니라, 사용자의 조건을 서비스 전체 추천과 알림으로 연결하기 위한 기반입니다.


[10번 슬라이드] DB 연동 화면 04

금융 프로필에서는 보유 자산, 연 소득, 월 저축액 같은 정보를 관리합니다.

이 데이터는 대출 계산과 LH 추천에 활용됩니다. 즉, 사용자가 입력한 조건을 바탕으로 어떤 공고나 매물이 현실적으로 가능한지 판단하는 데 사용됩니다.

공지사항 화면은 관리자가 등록한 서비스 점검이나 안내 내용을 테이블 형태로 보여주는 기능입니다.


[11번 슬라이드] 실거래 매물

이 화면은 HOME FIT의 가장 기본이 되는 실거래 매물 탐색 기능입니다.

SSAFY에서 제공받은 아파트 실거래 데이터를 기반으로 하고, 여기에 공공데이터 포털에서 가져온 오피스텔과 원룸 실거래 데이터까지 함께 연결해서 사용자가 다양한 주거 유형을 한 번에 확인할 수 있도록 구성했습니다.

기존에는 아파트는 아파트대로, 원룸이나 오피스텔은 다른 플랫폼에서 따로 찾아봐야 했는데, HOME FIT에서는 지도 위에서 지역을 기준으로 거래 정보를 확인하고, 주거 유형별로 비교할 수 있습니다.

기술적으로는 프론트에서 TanStack Query를 사용해서 시도, 구군, 동, 그리고 유형별 실거래 요청을 캐싱합니다. 같은 지역을 반복 조회할 때 매번 서버에 새 요청을 보내지 않고, 필요한 시점에만 데이터를 갱신하도록 설계했습니다.

백엔드에서는 실거래 데이터를 한 번 조회할 때마다 외부 API에 의존하기보다, 배치 구조를 통해 공공데이터를 정제해서 저장하고, 사용자는 저장된 데이터를 빠르게 조회하는 방향으로 구성했습니다.


[12번 슬라이드] 대출 상품 & 월 납부 계산

이 기능은 단순히 "이 집이 얼마인지"를 보는 데서 끝나는 것이 아니라, 사용자가 실제로 감당할 수 있는 집인지 판단할 수 있도록 만든 기능입니다.

금융감독원 오픈 API를 통해 주택담보대출과 전세자금대출 관련 상품 정보를 가져오고, 사용자가 입력한 보유 자산, 연 소득, 월 가용 금액을 기준으로 예상 월 이자 부담을 계산합니다.

예를 들어 매물 가격에서 보유 자금을 제외한 필요 대출 금액을 계산하고, 상품별 금리 조건을 적용해서 월 납부액이 사용자의 월 가용 금액 안에 들어오는지 판단할 수 있게 했습니다.

현재는 완성도를 계속 다듬는 단계지만, 핵심 의도는 실거래 탐색과 금융 계산을 분리하지 않고 하나의 흐름으로 연결하는 것입니다.

기술적으로는 금융감독원 API에서 가져온 대출 상품을 백엔드 배치 서비스로 저장하고, 프론트에서는 TanStack Query와 mutation을 이용해 금융 프로필 저장, 대출 분석 요청, 결과 갱신을 관리합니다.


[13번 슬라이드] LH 공고

이 화면은 공공데이터에서 가져온 공공임대와 LH 공고를 한눈에 볼 수 있도록 만든 기능입니다.

사용자는 지역, 유형, 모집 상태를 기준으로 공고를 확인할 수 있고, 로그인한 뒤에는 관심 공고로 저장할 수 있습니다. 이후 마이데이터에 입력한 소득, 자산, 월 저축액 같은 조건을 활용해서 사용자에게 더 적합한 공고를 추천하는 흐름으로 이어집니다.

또한 조건에 맞는 공고나 관심 등록한 공고가 있을 경우 이메일을 발송할 수 있도록 구성했습니다. 특히 신청 기간이 임박한 공고를 사용자가 직접 기억하지 않아도 알림을 받을 수 있게 하는 것이 목적입니다.

기술적으로는 LH 공고 데이터를 스케줄러와 배치를 통해 주기적으로 수집하고, DB에 캐싱해 둔 데이터를 사용자에게 제공합니다. 이메일은 Spring Scheduler가 정해진 시간에 관심 공고와 추천 공고를 검사하고, JavaMailSender를 통해 발송하는 구조입니다.

프론트에서는 TanStack Query로 LH 공고 목록, 상세, 관심 등록, 추천 공고 데이터를 관리하고, 관심 등록 이후에는 관련 쿼리를 invalidate해서 화면 상태가 바로 갱신되도록 했습니다.


[14번 슬라이드] Mail Service

메일 서비스는 HOME FIT에서 사용자가 공고를 놓치지 않게 만드는 알림 기능입니다.

관심 공고를 저장하거나 추천 메일 수신을 허용한 사용자를 대상으로, 공고 상태와 마감일을 확인해서 필요한 시점에 이메일을 보낼 수 있도록 구성했습니다.

예를 들어 마감이 임박한 공고, 오늘 마감인 공고, 또는 사용자의 마이데이터 조건과 잘 맞는 추천 공고를 이메일로 전달합니다.

이 기능은 Spring Scheduler와 JavaMailSender를 기반으로 구현했고, 중복 발송을 막기 위해 이메일 발송 이력도 함께 관리하는 방향으로 설계했습니다.


[15번 슬라이드] LH 캘린더

LH 공고는 단순히 목록만 확인하는 것으로는 부족하다고 생각했습니다. 공고마다 접수 시작일, 진행 중인 기간, 마감일이 다르고, 사용자가 여러 공고를 동시에 보면 일정을 놓치기 쉽기 때문입니다.

그래서 앞서 설명한 LH 공고들을 캘린더 형태로 한눈에 볼 수 있게 만들었습니다. 사용자는 월간 캘린더에서 접수 시작, 공고 진행 중, 마감 일정을 날짜 기준으로 확인할 수 있습니다.

이 기능은 특히 공공임대를 준비하는 사용자에게 중요합니다. 좋은 공고를 찾는 것만큼, 신청 기간을 놓치지 않는 것도 실제 주거 선택에서 큰 영향을 주기 때문입니다.

기술적으로는 백엔드에서 배치로 수집한 LH 공고의 일정 데이터를 저장하고, 프론트에서는 TanStack Query로 공고 목록을 받아 캘린더 이벤트로 가공합니다. 또한 스케줄러 기반 이메일 알림과 연결해서 마감이 가까운 공고를 사용자에게 알려주는 흐름까지 확장했습니다.


[16번 슬라이드] 양도 게시판

양도 게시판은 빠르게 집을 구해야 하는 사용자를 위해 고안한 기능입니다.

대학생 추가 합격, 인턴 합격, 갑작스러운 근무지 이동처럼 짧은 시간 안에 거주지를 찾아야 하는 경우가 많습니다. 하지만 일반적인 부동산 계약은 기본 기간이 6개월 이상이거나 초기 비용이 커서, 단기간 거주가 필요한 사람에게는 부담이 됩니다.

그래서 HOME FIT에서는 보증금, 월세, 관리비, 양도비, 입주 가능일, 남은 계약 기간을 한눈에 비교할 수 있는 양도 게시판을 제공했습니다. 사용자는 빠른 입주가 가능한 매물을 조건 중심으로 탐색할 수 있습니다.

이 기능에서 S3는 게시글 이미지 저장에 사용됩니다. 사용자가 양도 매물을 등록할 때 이미지를 서버에 직접 쌓아두는 것이 아니라, S3에 저장하고 DB에는 이미지 URL을 연결하는 방식으로 확장성을 고려했습니다.

프론트에서는 양도 게시글 목록, 상세, 작성, 수정, 삭제, 관심 등록을 TanStack Query의 useQuery와 useMutation으로 관리하고, 등록이나 수정 이후 관련 목록을 갱신하도록 구성했습니다.


[17번 슬라이드] 생활권 분석

생활권 분석은 사용자가 단순히 집 안의 조건만 보는 것이 아니라, 실제로 그 동네에서 생활할 때 편한지 판단할 수 있도록 만든 기능입니다.

카카오 지도 API를 통해 주변 편의시설을 조회하고, 병원, 마트, 약국, 카페, 학교 같은 생활 인프라를 반경별로 보여줍니다. 사용자는 500m, 1km, 1.5km처럼 범위를 선택해서 주변 환경을 확인할 수 있습니다.

또한 Open API와 지오코딩을 활용해서 버스 정류장과 지하철역 정보도 함께 확인할 수 있도록 했습니다. 단순히 "근처에 뭐가 있다"가 아니라, 교통 접근성과 생활 편의성을 함께 판단하는 구조입니다.

백엔드에서는 카카오 로컬 API와 교통 관련 공공데이터를 연동하고, 버스 정류장 정보는 스케줄러와 배치 구조로 미리 수집해 조회 성능을 높이는 방향으로 설계했습니다.

프론트에서는 생활권 분석 요청을 TanStack Query로 관리해서, 사용자가 반경이나 주소를 바꿨을 때 필요한 데이터만 다시 불러오도록 했습니다.


[18번 슬라이드] 사용자 이용 흐름

사용자 이용 흐름은 네 단계로 정리할 수 있습니다.

첫 번째, 사용자는 매매나 장기 거주, 단기 거주, LH 공공임대 중 자신의 목적을 선택하고 조건을 설정합니다.

두 번째, HOME FIT에서 아파트, 오피스텔, 원룸 실거래가와 양도 매물, LH 공고를 조회하고 주변 생활환경을 분석합니다.

세 번째, 대출 상품 조회와 월 납부 계산으로 자금 부담을 확인하고, 관심 매물이나 공고를 저장해서 이메일 알림을 받을 수 있습니다.

마지막으로 탐색, 생활 분석, 자금 계산, 일정 관리가 하나의 흐름으로 연결되어 최종 주거 선택을 돕습니다.


[19번 슬라이드] Why HOME FIT

HOME FIT의 차별점은 네 가지입니다.

첫 번째는 다양한 주거 유형 통합입니다. 아파트뿐 아니라 오피스텔, 원룸, 다가구 주택까지 한 지도에서 비교할 수 있습니다.

두 번째는 주거 탐색과 금융 계산의 연결입니다. 실거래 정보를 본 뒤 대출 상품과 월 납부 금액까지 확인해서, 감당 가능한 주거 수준을 판단할 수 있습니다.

세 번째는 장기 주거와 단기 주거를 함께 지원한다는 점입니다. 공식 실거래 정보와 LH 공고뿐 아니라 사용자 양도 매물까지 하나의 서비스에서 제공합니다.

네 번째는 공고 검색 이후까지 관리한다는 점입니다. 관심 등록, 캘린더, 마감 이메일, 마이데이터 기반 추천까지 이어져서 사용자가 중요한 일정을 놓치지 않도록 돕습니다.


[20번 슬라이드] System Architecture

기술 구조를 간단히 정리하면, HOME FIT은 Vue 3 프론트와 Spring Boot 백엔드로 구성되어 있습니다.

사용자가 화면에서 데이터를 요청하면 Vue 3에서 TanStack Query가 먼저 캐시를 확인하고, 필요한 경우 Axios를 통해 Spring Boot API를 호출합니다.

백엔드에서는 Spring Security와 JWT로 사용자 인증을 처리하고, Service 계층과 MyBatis를 통해 MySQL 데이터를 조회합니다.

공공데이터는 사용자가 요청할 때마다 외부 API를 직접 호출하는 방식이 아니라, Scheduler와 Batch를 통해 LH 공고, 실거래, 대출 상품, 버스 정류장 정보를 주기적으로 수집하고 정제한 뒤 DB에 저장하는 구조입니다.

이렇게 구성한 이유는 외부 API 응답 속도나 호출 제한에 영향을 덜 받고, 사용자는 더 안정적으로 데이터를 조회할 수 있게 하기 위해서입니다.


[21번 슬라이드] Tech Stack & Features

핵심 기술 스택은 JWT와 Spring Security, Spring Batch와 Scheduler, Spring AI, TanStack Query입니다.

JWT와 Spring Security는 관심 공고, 마이데이터, 양도 게시글처럼 사용자별로 보호되어야 하는 기능을 안전하게 관리하기 위해 사용했습니다.

Spring Batch와 Scheduler는 LH 공고, 오피스텔, 원룸 실거래, 대출 상품, 버스 정류장 같은 외부 공공데이터를 자동으로 수집하고 정제하는 역할을 합니다.

Spring AI는 마이데이터 기반 LH 공고 추천이나 추천 이메일 생성처럼, 사용자 조건과 공고 내용을 의미적으로 비교하는 확장 구조로 활용할 수 있습니다.

TanStack Query는 프론트에서 서버 데이터를 관리하는 핵심입니다. 실거래 지도, LH 공고, 관심 등록, 마이페이지 데이터에서 캐싱과 중복 요청 방지, mutation 이후 화면 갱신을 담당합니다.

또한 양도 게시판 이미지 저장에는 S3를 사용해 서버 저장소 부담을 줄이고, 이미지 URL 기반으로 게시글과 연결되도록 구성했습니다.


[22번 슬라이드] 기대 효과 및 향후 발전 방향

HOME FIT의 기대 효과는 크게 네 가지입니다.

첫 번째는 탐색 효율입니다. 아파트, 오피스텔, 원룸을 한 번에 비교할 수 있고, 대학생이나 인턴처럼 단기 주거가 필요한 사용자의 탐색 시간을 줄일 수 있습니다.

두 번째는 생활환경과 공고 관리입니다. 이사 전 주변 편의시설과 교통환경을 확인하고, LH 공고를 캘린더와 이메일로 관리할 수 있습니다.

세 번째는 기술적 안정성입니다. 배치 수집으로 공공데이터를 안정적으로 제공하고, 프론트 캐시를 활용해 반복 조회 경험을 개선합니다.

마지막으로 향후에는 실거래가 변화 알림, 추천 모델 고도화, 대중교통 통근 시간 분석, 전세 사기 위험 정보 연계, 모바일 앱이나 PWA 확장까지 발전시킬 수 있습니다.


[23번 슬라이드] 마무리

정리하겠습니다.

HOME FIT은 집을 찾는 순간부터 실제로 살아갈 수 있는지 판단하는 순간까지, 실거래, 단기 주거, 공공임대, 생활환경, 금융 정보를 하나의 흐름으로 연결하는 서비스입니다.

아직 완성도를 더 높여야 할 부분은 있지만, 핵심 목표는 명확합니다. 사용자가 흩어진 정보를 직접 찾아다니지 않고, 자신의 조건에 맞는 주거 선택을 더 빠르고 합리적으로 할 수 있게 만드는 것입니다.

이상으로 HOME FIT 발표를 마치겠습니다. 감사합니다.
