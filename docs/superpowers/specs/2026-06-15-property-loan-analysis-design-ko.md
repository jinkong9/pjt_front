# 매물 상세 및 대출 분석 설계서

## 목표

실거래 지도에서 다른 페이지로 이동하지 않고 선택한 매물의 상세정보와 대출 분석을 확인한다. 검색 결과 옆에 고정 상세 패널을 열고, 상세정보·관심매물·회원 금융 프로필·보수적인 대출 준비도 분석을 제공한다.

이번 기능은 사용자가 직접 입력한 금융정보를 사용하며 실제 금융기관 마이데이터 연동은 범위에서 제외한다.

## 저장소와 브랜치

- 프론트엔드: `pjt_front`, `byeg` 브랜치
- 백엔드: `pjt_back`, `byeg` 브랜치
- 프론트엔드: Vue 3, Pinia, Axios, Vitest
- 백엔드: Spring Boot, 세션 인증, MyBatis, JUnit

## 사용자 화면

### 데스크톱

승인한 고정 3단 구조를 사용한다.

1. 검색 조건 및 실거래 결과 목록
2. 선택한 매물의 상세 패널
3. Kakao 지도

결과 카드의 `관심매물 등록` 버튼은 `상세보기`로 변경한다. 카드, 상세보기 버튼, 지도 마커를 선택하면 페이지 이동 없이 같은 상세 패널의 내용만 교체한다.

### 모바일

상세 패널은 지도와 검색 화면 위에 전체 높이 드로어로 표시한다. 닫아도 검색 결과와 조건은 유지한다.

### 상세정보 탭

- 아파트명
- 거래금액
- 주소
- 전용면적
- 층
- 거래일
- 생활권 분석 이동
- 관심매물 상태와 등록·해제 버튼

관심매물 등록은 로그인이 필요하다. 비로그인 사용자는 로그인 화면으로 안내한다.

### 대출 계산 탭

1. 비로그인: 로그인 안내
2. 로그인했지만 금융 프로필 없음: 탭 안에 입력 폼 표시
3. 금융 프로필 있음: 선택한 매물의 대출 분석 결과 표시

프로필을 저장하면 즉시 분석을 다시 실행한다. 같은 정보는 마이페이지에서도 조회하고 수정할 수 있다.

## 회원 금융 프로필

회원별로 다음 값을 한 건 저장한다.

- 보유자산
- 연소득
- 월 가용저축액
- 기존 대출잔액
- 기존 월 상환액

모든 금액은 원 단위의 0 이상 `DECIMAL`로 저장한다. `member_financial_profiles` 테이블은 `user_id`를 기본키로 사용하고 `members.user_id`를 참조한다.

## 백엔드 API

### 금융 프로필

- `GET /api/members/me/financial-profile`
  - 프로필이 있으면 `200`
  - 없으면 `204`
  - 비로그인이면 `401`
- `PUT /api/members/me/financial-profile`
  - 현재 로그인 회원의 프로필을 검증 후 저장 또는 갱신

### 매물 대출 분석

- `POST /api/loans/property-analysis`

요청값:

- `dealNo`
- `years`
- `repaymentType`
- 선택한 연 금리(선택)

백엔드는 세션 회원의 금융 프로필과 `dealNo`에 해당하는 실제 거래금액을 직접 조회한다. 프론트가 보낸 회원 ID, 자산, 소득, 매물 가격은 신뢰값으로 사용하지 않는다.

응답 영역:

- 매입비용 요약
- 대출 한도
- 자금 준비도
- 상환 계획
- 추천 대출상품
- 적용 가정과 고지문

기존 `/api/loans/products`, `/api/loans/calculate` API는 유지한다.

## 계산 기준

첫 버전은 대출 승인 판단이 아닌 보수적인 자금계획 시뮬레이션이다.

### 총 필요자금

- 매물 가격: 선택 거래의 실거래금액
- 예상 부대비용: 매물 가격 × 설정된 부대비용 비율
- 총 필요자금: 매물 가격 + 예상 부대비용
- 현재 보유자산: 회원 금융 프로필 값

부대비용은 취득세, 중개비 등 관련 비용을 단순화한 추정치이며 적용 비율을 결과에 표시한다.

### 대출 한도

- 자금 필요액: `max(총 필요자금 - 보유자산, 0)`
- LTV 한도: `매물 가격 × 보수적 LTV 비율`
- 연간 DSR 상환한도: `연소득 × DSR 비율`
- 신규대출에 사용할 수 있는 연간 상환액:
  `max(연간 DSR 상환한도 - 기존 월 상환액 × 12, 0)`
- DSR 대출한도: 선택 기간과 금리로 위 상환액을 감당할 수 있는 원금
- 예상 최대 대출액:
  `min(자금 필요액, LTV 한도, DSR 대출한도)`

기존 대출잔액은 결과에 표시한다. 기존 대출의 기간과 금리를 임의로 가정하지 않기 위해 DSR 차감에는 사용자가 입력한 기존 월 상환액을 직접 사용한다.

### 자금 준비도

- 최소 필요 자기자금:
  `max(총 필요자금 - min(LTV 한도, DSR 대출한도), 0)`
- 추가로 필요한 자산:
  `max(최소 필요 자기자금 - 보유자산, 0)`
- 목표 도달 예상기간:
  `ceil(추가 필요자산 / 월 가용저축액)`

추가 필요자산이 0이면 현재 가정상 구매 가능한 상태로 표시한다. 추가 자산이 필요하지만 월 가용저축액이 0이면 예상기간은 계산할 수 없다고 표시한다.

### 상환 계획

기존 `LoanCalculator`를 재사용해 예상 월 상환액, 첫·마지막 달 상환액, 총이자와 총상환액을 계산한다.

상환 후 월 여유금액은 `월 가용저축액 - 신규대출 예상 월 상환액`으로 표시하며, 가처분소득이 아닌 단순 현금흐름 참고값임을 알린다.

### 상품 추천

기존 FinLife 주택담보대출 상품을 다음 기준으로 정렬한다.

1. 숫자 금리 옵션 존재 여부
2. 적용 가능한 평균 또는 최저 금리가 낮은 순서
3. 메타데이터가 허용하는 범위에서 선택한 상환방식과의 적합도

추천에는 금융회사, 상품명, 금리, 상환 관련 정보, 원본 한도 설명을 포함한다. 문자열로 제공되는 상품 한도는 실제 승인 가능한 숫자 한도로 사용하지 않는다.

상품 조회에 실패해도 핵심 계산 결과는 반환하고 추천 목록과 경고만 별도로 표시한다.

## 설정값

백엔드 설정으로 다음 값을 관리하고 결과에 실제 적용값을 반환한다.

- 보수적 LTV 비율
- 보수적 DSR 비율
- 예상 부대비용 비율
- 기본 대출기간
- 최대 추천상품 개수

## 프론트엔드 구조

현재 큰 `PricesPage.vue`를 다음 책임으로 분리한다.

- 매물 검색·결과 패널
- 매물 상세 패널
- 상세정보 탭
- 금융 프로필 폼
- 대출 분석 결과

페이지는 선택 매물, 패널 열림 상태, 지도 연동을 관리한다. API별 로딩과 오류 상태는 해당 탭 가까이에서 관리한다. 마이페이지에도 동일한 금융 프로필 입력 영역을 추가한다.

## 상태 및 오류 처리

- 새 검색 후 첫 매물을 자동으로 열지 않는다.
- 상세 닫기는 선택만 해제한다.
- 다른 매물을 선택하면 이전 분석을 무효화하고 새 거래번호로 다시 분석한다.
- 로그인 회원의 관심매물 상태를 조회하고 토글 후 갱신한다.
- `401`: 로그인 안내
- 금융 프로필 `204`: 입력 폼
- 검증 오류: 입력값 유지 및 필드 메시지 표시
- 상품 API 오류: 계산 결과 유지 및 추천 오류만 표시
- 지도 SDK 오류: 검색·상세·관심매물·대출 분석은 계속 사용 가능

## 보안과 개인정보

- 프로필 API는 세션에서 회원 ID를 결정한다.
- 프론트는 프로필 요청에 회원 ID를 보내지 않는다.
- 분석 API는 `dealNo`로 DB의 거래금액을 직접 읽는다.
- 로그에 전체 금융 프로필을 남기지 않는다.
- 금융 프로필은 다른 회원이나 관심매물 응답에 노출하지 않는다.

### 프론트엔드

- 결과 목록의 `상세보기`
- 상세 패널 열기·닫기·매물 교체
- 상세정보·대출 계산 탭 전환
- 관심매물 토글과 비로그인 안내

- 마이페이지 프로필 수정

## 완료 조건

- `/prices`에서 페이지 이동 없이 매물을 확인한다.
- 검색 결과, 상세 패널, 지도가 같은 선택 상태를 공유한다.
- 상세정보 탭에서 관심매물을 등록·해제한다.
- 로그인 회원이 금융 프로필을 저장·수정한다.
- 대출 탭에서 총 필요자금, 보유자산, 예상 최대 대출액, 최소 자기자금, 추가 필요자산, 목표기간, 상환액, 추천상품을 확인한다.
- 적용한 계산 가정과 참고용 시뮬레이션 고지문을 표시한다.
- Java 17 백엔드와 Vite 프론트가 실행되고 `/api` 프록시를 통해 통신한다.
