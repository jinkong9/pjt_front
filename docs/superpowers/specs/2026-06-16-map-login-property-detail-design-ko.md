# 지도, 로그인 복귀, 매물 상세정보 개선 설계

## 목표

- Kakao 지도 SDK가 `localhost:8080`과 `localhost:5173`에서 정상 로드되도록 Web 플랫폼 도메인을 등록한다.
- 일반 로그인은 메인 화면으로 이동하고, 매물 상세에서 시작한 로그인은 기존 검색 결과와 선택 매물로 복귀한다.
- 로그인 화면을 전역 버튼 스타일의 영향에서 분리해 데스크톱과 모바일에서 안정적으로 표시한다.
- 매물 상세정보 탭에서 거래 정보뿐 아니라 생활권 점수, 상권, 대중교통, 교통안전 정보를 한 화면의 연속 스크롤로 제공한다.

## Kakao 지도

현재 JavaScript 키는 `/api/config`에서 정상 전달되지만 Kakao SDK가 `domain mismatched`로 요청을 거절한다. Kakao Developers에서 해당 애플리케이션의 Web 플랫폼 사이트 도메인에 다음 두 주소를 등록한다.

- `http://localhost:8080`
- `http://localhost:5173`

프론트엔드는 SDK 오류를 단순 키 오류로만 표시하지 않고, 도메인 등록이 필요한 경우 확인할 주소를 안내한다. 키 자체는 저장소에 새로 기록하지 않고 기존 백엔드 환경변수를 계속 사용한다.

## 로그인 이동 규칙

일반 메뉴에서 로그인한 경우 성공 후 `/home`으로 이동한다.

매물 상세의 관심 매물 또는 대출 계산 안내에서 로그인한 경우 로그인 링크에 `redirect` 쿼리를 넣는다. 값에는 현재 `/prices`의 검색 쿼리와 선택한 매물 번호, 열려 있던 탭을 포함한다.

예시:

```text
/login?redirect=/prices?...&trade=7928647&tab=loan
```

로그인 성공 후 프론트는 내부 경로로 검증된 `redirect`만 사용한다. 외부 URL이나 `//`로 시작하는 값은 거부하고 `/home`으로 이동한다. 복귀한 실거래 지도는 검색 결과를 다시 조회한 후 해당 매물을 선택하고 기존 탭을 연다.

## 로그인 화면

로그인 페이지 전용 클래스와 scoped 스타일을 사용한다.

- 제출 버튼은 붉은 배경과 흰 글자를 명시한다.
- 카드의 최대 너비와 최소 높이를 화면 크기에 맞춰 조정한다.
- 헤더 아래 여백과 카드의 세로 정렬을 고정한다.
- 작은 화면에서는 환영 패널과 입력 폼을 한 열로 배치한다.
- OAuth 버튼은 기존 색상을 유지하되 공통 `button` 규칙보다 높은 우선순위를 갖는다.

## 상세정보 구성

상세정보 탭은 A안인 단일 연속 스크롤을 사용한다.

1. 매물 기본 정보
   - 거래 가격
   - 전용면적
   - 층
   - 거래일
2. 생활권 종합 평가
   - 종합 점수와 등급
   - 상권 점수
   - 대중교통 점수
   - 교통안전 점수
   - 백엔드 분석 메시지
3. 상권 요약
   - 전체 시설 수
   - 음식, 카페, 의료, 편의, 생활 시설 수
4. 대중교통 요약
   - 500m와 1km 이내 지하철 수
   - 300m, 500m, 1km 이내 버스정류장 수
5. 교통 위험 요약
   - 교통 이벤트 수
   - 도로공사 수
   - 위험 등급
6. 가까운 생활 시설
   - API 응답 전체를 무제한으로 펼치지 않고 우선 5건을 표시한다.
   - 더보기로 나머지 시설을 펼칠 수 있다.
7. 교통 체크 포인트
   - 우선 3건을 표시하고 더보기로 나머지 이벤트를 펼친다.
8. 관심 매물 등록과 생활권 분석 전체화면 링크

상세 패널이 열리면 매물 좌표와 주소를 사용해 기존 `/api/analysis`를 호출한다. 매물이 바뀌면 이전 결과를 초기화하고 새 분석을 요청한다.

## 상태 및 오류 처리

- 분석 요청 중에는 상세 탭 안에 스켈레톤 또는 로딩 문구를 표시한다.
- 분석 API가 실패해도 가격과 거래 정보, 관심 매물 기능은 계속 사용할 수 있다.
- 분석 실패 시 가짜 점수를 만들지 않고 재시도 버튼과 오류 안내를 표시한다.
- 좌표가 없는 매물은 생활권 분석을 요청하지 않고 분석 불가 사유를 표시한다.
- 매물 변경 중 이전 매물의 느린 응답이 도착해도 현재 매물 결과를 덮어쓰지 않도록 요청 식별 또는 취소 처리를 적용한다.

## 테스트

- 일반 로그인 성공 후 `/home` 이동
- 유효한 내부 `redirect` 복귀와 외부 URL 차단
- 매물 상세 로그인 링크에 현재 검색 경로, 매물 번호, 탭 포함
- 복귀 후 선택 매물 및 상세/대출 탭 복원
- 상세 패널의 `/api/analysis` 호출 파라미터 검증
- 점수, 상권, 대중교통, 교통안전 데이터 렌더링
- 시설과 교통 이벤트 더보기 동작
- 분석 실패 및 좌표 누락 상태
- 로그인 화면의 전용 클래스와 버튼 스타일 회귀 테스트
- 8080 통합 빌드와 5173 개발 서버에서 지도 및 로그인 복귀 수동 검증
