# 백엔드 제공 SPA 및 대출 UI 보정 설계

## 목표

- 개발 중에는 `http://localhost:5173`의 Vite 프론트가 `/api` 요청을 `8080` 백엔드로 전달한다.
- 통합 실행에서는 `http://localhost:8080`이 같은 Vue 빌드를 제공하고 API도 동일 출처에서 호출한다.
- 기존 Thymeleaf 화면 대신 Vue Router 경로를 새로고침해도 SPA가 열린다.
- 대출 상세 패널 버튼과 결과 카드가 전역 버튼 CSS에 덮이지 않고 일관된 화면을 유지한다.

## 배포 구조

- `pnpm build:backend`는 Vue 앱을 `pjt_back/src/main/resources/static/app`에 생성한다.
- 빌드된 `index.html`은 `/app/assets/...` 경로를 사용한다.
- 백엔드 필터는 화면 GET 경로를 `/app/index.html`로 전달한다.
- `/api`, `/uploads`, `/swagger-ui`, `/v3/api-docs`, `/app` 정적 파일은 전달 대상에서 제외한다.
- Vue Router의 정적 경로와 동적 상세 경로를 지원한다.

## UI 보정

- 상세 패널 내부 버튼은 전용 클래스로 배경·테두리·높이를 명시한다.
- 비활성 탭은 흰색 배경과 중립색 글자, 활성 탭은 옅은 붉은 배경과 붉은 글자로 표시한다.
- 닫기 버튼은 흰색 배경의 작은 아이콘 버튼으로 변경한다.
- 대출 결과 상단은 검정 블록 대신 옅은 상태 카드로 변경한다.
- 금액 카드에는 총 필요자금, 보유자산, 최대 대출액, 최소 자기자금, 추가 필요자금을 구분해 표시한다.
- 음수 월 여유금액은 `월 상환 부담 초과` 상태와 붉은 색상으로 표시한다.

## 검증

- 백엔드 테스트에서 `/`, `/prices`, 동적 Vue 경로가 Vue `index.html`을 반환하는지 확인한다.
- 프론트 컴포넌트 테스트에서 탭·닫기 버튼의 전용 스타일과 추가 필요자금 표시를 확인한다.
- `5173`과 `8080` 양쪽에서 실거래 조회와 상세 패널을 브라우저로 확인한다.
